## 为什么开这个博客

我是**程序员不鸭**，做 Java 后端，最近几年在把 AI Agent 往生产里落。

开这个博客的原因很简单：**踩过的坑如果不写下来，下次还会踩一遍**。

## 这里会有什么

- **Java 后端**：Spring Boot / Spring Cloud、并发与线程模型、JVM 调优、Redis 缓存设计、消息中间件、稳定性与可观测性。
- **AI Agent**：LangChain / LangGraph / Spring AI、Agent 编排、RAG 检索增强、Function Calling、Prompt 工程、Agent 落地的工程约束。
- **基础设施**：Docker、Kubernetes、CI/CD。

一句话立场：**Agent 不是聊天机器人，是一个能调工具、有状态、可工程化的系统**。Java 这套成熟的工程方法论，正是 Agent 从 Demo 走向生产必须补上的那一课。

## 写作原则

- 不写水文，只写自己验证过的东西。
- 先讲清楚问题，再讲方案，最后讲为什么是这个方案。
- 代码能跑就贴能跑的，不能跑就说明为什么。
